# Case Theory

## 목표
이 사건을 단순 사칭 신고에서 끝내지 않고, 자본시장 불공정거래 제보로 발전시키려면 어떤 논리 구조가 필요한지 정리한다.

## 최소 논리 구조
### Layer 1: 허위 신뢰 형성
- 사이트가 공공기관/금융감독기관/금융권 공식 프로젝트처럼 보이게 함
- 이로 인해 일반 투자자가 공식성/합법성을 오인할 가능성이 생김

### Layer 2: 투자유인
- 예치, 상품가입, 수익금, 투자현황 등으로 금전 유입 구조를 형성함
- 투자자에게 자금 투입을 유도함

### Layer 3: 자본시장 연결
이 단계가 있어야 포상금 트랙으로 의미가 커진다.
- 특정 상장주 추천
- 특정 종목/테마에 대한 허위 재료 유포
- 채널 운영자의 선행매매/사후매도
- 유료 리딩방/회원제 구조

## 강한 케이스 예시
1. 사이트 가입 유도 메시지에서 `케이뱅크`, `스테이블`, `정부 프로젝트`, `급등` 같은 표현이 나옴
2. 같은 채널에서 상장주 종목명과 매수 시점이 제시됨
3. 추천 직후 급등하거나 운영자가 매도한 정황이 남아 있음

## 약한 케이스
- 사이트만 존재
- 기관 사칭만 보임
- 종목 언급 없음
- 투자금 수취만 있고 주식 연결 없음

## 실무 원칙
- 증거 없는 연결은 쓰지 않는다.
- 대신 `허위 신뢰 형성 -> 투자유인 -> 상장주 연결 가능성` 구조로 단계별로 쓴다.
- 자본시장 연결 증거가 생기면 그때 바로 불공정거래 제보 문안을 강화한다.
